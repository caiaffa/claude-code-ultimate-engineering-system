# ADR_REVIEW_CHECKLIST

Use this checklist to review ADR quality.

## Problem quality
- [ ] The problem is stated clearly
- [ ] Symptoms are not confused with root cause
- [ ] Business relevance is explicit
- [ ] Cost of inaction is described

## Option quality
- [ ] More than one realistic option was considered
- [ ] Alternatives were evaluated honestly
- [ ] Trade-offs are explicit
- [ ] Operational cost was considered
- [ ] Security impact was considered
- [ ] Observability impact was considered
- [ ] Reversibility was considered

## Decision quality
- [ ] The chosen option is justified against constraints
- [ ] Hidden assumptions are named
- [ ] Risks are concrete, not vague
- [ ] Success criteria are measurable
- [ ] Rollout and rollback are credible

## Anti-handwaving
- [ ] “Scalable”, “robust”, and “simple” are backed by mechanisms
- [ ] “Best option” is tied to an explicit objective function
- [ ] “Production-ready” is justified with operational safeguards
